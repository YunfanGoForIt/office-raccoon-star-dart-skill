# Star-DART 每周回顾 PPT 模板

## 目标

每周 PPT 用于帮助用户复盘本周 Star 收藏：不是项目逐条堆砌，而是把“本周看到什么、哪些值得继续研究、能沉淀成什么能力”讲清楚。

## 推荐页结构

| 页码 | 页面 | 内容 |
| --- | --- | --- |
| 1 | 封面 | `Star-DART 周回顾｜YYYY-Wxx`，本周新增 Star 数、时间范围 |
| 2 | 总览 | 新增项目总数、技术方向分布、复用等级分布、最值得关注项目 |
| 3 | Top 3 项目 | 每个项目一句话定位、为什么值得 Star、推荐动作 |
| 4 | 技术方向地图 | 按 AI Agent / 开发工具 / Awesome List / 前端可视化 / 其他分类 |
| 5 | 复用灵感 | 可以转化为 Skill、Agent、工作流、产品功能或内容选题的想法 |
| 6 | 观察池 | 信息不足、维护风险、License 风险或需要实际运行验证的项目 |
| 7 | 下周动作 | 优先阅读、优先试跑、优先沉淀、暂缓观察 |
| 8 | 结束页 | 一句话总结本周收获 |

## 写作要求

- 少写长段落，多用图表、卡片、对比和大数字。
- 每页必须有一个清晰结论，不能只是列表。
- 只使用本周 Base 台账和项目文档中已有的信息，不编造项目数据。
- 对信息不足的项目，明确标记为“待确认”或“观察池”。
- PPT 用于强化记忆，所以要突出“为什么我当时 Star 它”和“下一步怎么复用”。

## 推荐视觉风格

- 风格：技术雷达、知识资产、轻量情报周报。
- 配色：深色科技底或干净浅色卡片均可，但全 deck 必须统一。
- 页面元素：指标卡、分类图、Top 3 卡片、行动清单、风险标签。

# Star-DART 项目文档模板

## 文档目标

这份文档不是 README 翻译，而是帮助未来的自己快速判断：

- 这个项目解决什么问题？
- 为什么当时值得 Star？
- 它能不能被复用到自己的 Agent、Skill、内容、产品或研发流程里？
- 如果要上手，最快路径是什么？

## 标准结构

````markdown
---
title: {中文标题}
repo: {owner/repo}
github_url: https://github.com/{owner}/{repo}
category: {技术方向}
generated_at: {时间戳}
---

# {中文标题}

- 资料中不明确的地方。
- 需要实际运行验证的地方。

## 🔗 相关资源

- GitHub: https://github.com/{owner}/{repo}
- 文档 / 官网：{如有}
- Release / Issues：{如有}
```

## 写作要求

- 不要逐句翻译 README。
- 不要把项目描述写成广告语。
- 不要编造 README 或公开资料中没有的信息。
- 优先解释“为什么值得 Star”和“未来怎么复用”。
- 技术术语、函数名、变量名、包名保留英文。
- 如果项目资料不足，明确写入“待确认”，不要硬凑。

- [ ] 包含边界与待确认事项。
- [ ] 有 GitHub URL。
- [ ] 文档标题格式为 `{中文标题} - {owner/repo}`。
